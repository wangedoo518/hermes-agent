# Tencent Meeting Chrome/CDP Import Design

Date: 2026-05-23

## Decision

Use the official `tmeet` CLI as the primary Tencent Meeting exporter, and add a
Chrome/CDP importer as a supplemental route for web-only or rate-limited assets.

The CDP route is not a replacement for `tmeet auth login`. The Tencent Meeting
record center is a JavaScript SPA. When Chrome is not logged in on
`meeting.tencent.com`, the page redirects to `login.html`, and no record-list
API data is available to capture.

## Why CDP Is Useful

- It uses the same browser session that a human can see in the Tencent Meeting
  web UI.
- It can capture JSON responses generated by the page after scrolling, opening a
  record, or downloading assets.
- It avoids storing cookies or request headers in Obsidian.
- It gives us an evidence bundle when the CLI hits daily API limits.

## Page Findings

`https://meeting.tencent.com/user-center/meeting-record` returns a shell page and
loads the user-center app from Tencent CDN. The record center code is split into
lazy chunks such as `MyRecordList`, `MeetingRecordInfo`, and
`DownloadWemeetRecord`.

Useful web routes and API strings observed in the page bundles:

- `/user-center/meeting-record`
- `/user-center/shared-record-info`
- `/meeting-record/shares`
- `/v2/meetlog/main/clear-reddot`
- `/v2/meetlog/record-detail/get-recording-exportlink`
- `/v2/cloud-record/settings`
- `/v2/cloud-record/share`

The list page includes fields such as `meetingId`, `meetingCode`,
`uniRecordId`, `recordType`, `title`, `initialStartTime`, `recordCount`, and
`duration`.

## Implementation

Added:

- `scripts/tmeet_capture_meeting_record_page_cdp.py`
- Hermes tool: `tmeet_capture_meeting_record_page_cdp`
- Full pagination support through the visible record-list UI. Use
  `--max-pages 0` to visit all pages.

Output location:

```text
<wiki>/raw/tencent-meetings/cdp-captures/<timestamp>/
  capture_report.json
  source.md
  dom.md
  record_list.json
  record_list.md
  responses/
```

Latest manifest:

```text
<wiki>/_derived/tencent-meetings-cdp-latest.json
```

The capture filters responses to Tencent Meeting / WeMeet domains and relevant
record paths only. It intentionally excludes analytics endpoints such as Aegis
and Google Analytics.

## Operating Procedure

1. Start Chrome with remote debugging on port `9222`.
2. In that same Chrome profile, open
   `https://meeting.tencent.com/user-center/meeting-record`.
3. Complete Tencent Meeting web login.
4. Run:

```bash
cd /Users/champion/Documents/develop/hermes-agent
python scripts/tmeet_capture_meeting_record_page_cdp.py --wait-ms 15000 --json
```

To import the full visible record list:

```bash
python scripts/tmeet_capture_meeting_record_page_cdp.py \
  --url 'https://meeting.tencent.com/user-center/meeting-record?reload=1' \
  --wait-ms 5000 \
  --max-pages 0 \
  --json
```

If `requires_web_login` is `true`, the capture is valid as a diagnostic but does
not contain meeting records. Login in Chrome and run again.

## Boundaries

- Does not save cookies, request headers, or browser storage.
- Does not click destructive actions.
- Does not replace `tmeet_export_records_to_obsidian.py`.
- Does not bypass Tencent Meeting login; it uses the user's own logged-in
  browser session.

not logged into Tencent Meeting web. The script correctly wrote a capture bundle
and returned:

```json
  "json_response_count": 0
}
```

After logging into Tencent Meeting web in the same Chrome profile, full
