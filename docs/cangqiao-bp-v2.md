## 1. 一句话定位

**仓桥智能是一家「本体论驱动的 AI 落地交付公司（FDC）+ FDE 人才工厂」：用 Hermes Agent 工具链把行业本体（业务地图）和交付方法沉淀成可复用资产（项目仓），用真实项目把大学生训练成 AI FDE 交付军团（人才仓），让企业用得起 AI 落地，让年轻人长成 AI 时代的稀缺交付人才。**

两仓一桥：

- **项目仓**：行业本体（Ontology）+ 真实交付案例 + Skill/Agent 模板 + 效果数据。**每交付一次，本体厚一层，下次交付更便宜。**
- **人才仓**：大学生经 V1-V7 训练成 FDE（Echo+Delta），按等级承接项目仓任务。
- **桥**：把「企业缺落地交付」与「学生缺真实项目」对接，AI 做重复，人做判断、审美、客户与本体设计。

## 2. 市场痛点：从五层认知看 FDE 在中国的真问题

### 2.1 企业侧：AI 落不了地，是因为缺「本体」这层翻译

MIT NANDA 2025：**95% 企业生成式 AI 试点没有可衡量业务影响**。根因不是模型弱，是 L3 缺失——AI 直接面对企业几百张表、上千个字段，必然幻觉、必然不懂业务。企业要的不是聊天机器人，是「能解决问题的全流程服务」。

通用 AI 工具的死穴（对照会议 L1/L2）：

- 没有客户私有上下文、不懂业务语义（缺本体）。
- 拼人月外包做定制：贵、慢、易返工（L1 被做臭）。
- Echo+Delta 跨职能在国内难凑齐（L2 难复制）。

### 2.2 人才侧：FDE 是公认稀缺岗，但供给断层

19 个在招岗 + 公开行研共同指向：大模型应用人才缺口 **300-400 万**，企业「不缺会调模型的，缺能把 AI 真正用起来的人」。FDE 是 AI 落地最后一公里、由人来扛的角色，而中国几乎没有系统化的 FDE 供给来源。

### 2.3 学生侧：想进 AI、却卡在「没有真实项目」

2026 届毕业生 **1,270 万**（新高）；通杀筛选门槛是项目经验/实战，「普通简历过不了初筛」。**学生缺的正是一个能产出真实交付作品集的训练场。**

### 3.2 FDE 统一能力模型（从 19 个 JD 逆推 —— 课程的目标产物）

**通用核心（几乎每个 JD 都要）：**
- Echo/Delta 双角色（蚂蚁数科/字节/泰豪/上海P3P7 明确点名 Palantir）。
- 需求洞察：把「老板想用 AI」翻译成可执行 spec（**全部 JD**）。
- **Vibe Coding**：Claude Code/Codex/Cursor 快速原型（波起二象、智控、OpenClaw、字节明确）。
- Python 生产代码（+ TS/Java/Go）；API/数据库/系统集成。
- LLM 应用栈：提示词工程、Function Calling/Tool Use、RAG（切片/Embedding/向量库）、Agent（多 Agent 编排/Memory/Planning）、**Eval 评测**（中央部委/灯塔强调抓幻觉/grounding）。
- 全交付闭环：需求→PoC→原型→集成→验收→上线→迭代→运维。
- 沉淀复用：Skill 模板/Prompt 库/Agent 模板/SOP/方法论（**全部 JD**）。

**企业级进阶（高薪/政企岗才要）：**
- **本体论/Ontology/语义层**（灯塔、字节、阿里云知识图谱、OntoMind「组织级语义壁垒」、政企端到端、中央部委）——**这正是会议 L3，也是仓桥的产品核心。**
- SQL/数据治理/ETL/AI-Ready 数据层（字节、政企、灯塔）。
- 系统集成：MCP 打通 CRM/ERP/OA/HIS/EMR（腾讯云、智控医疗）。
- 私有化部署/数据合规/等保/脱敏/审计（智控、OntoMind、中央部委、腾讯云）。
- 推理部署 vLLM/TGI/Docker/K8s；微调 SFT/LoRA/QLoRA/DPO（阿里云、智控、灯塔）。

### 3.3 三个决定 BP 成败的关键发现

**发现一：市场已为仓桥的技术栈点名。**
- **JD #19 明确要「Hermes 二次开发」**；JD #10 腾讯云要「Agent、RAG、MCP、SDD、Harness」；JD #16 要「AI Agent Skill 技能组开发与交付」。**仓桥所有项目（lufei-profile/career-coach-copilot）就是 Hermes 上的 Skill/Agent 工件。学生在仓桥写的每个 Skill，就是这些 JD 逐条要的作品。**

**发现二：入口对学生敞开——FDE 市场已分叉。**
- 大厂要 5-10 年（学生进不来），但 **波起二象「经验不限、学历不限、按能力与作品定级」、智控医疗「不以工作年限为核心筛选、应届生欢迎、请携带你的 Vibe Coding 项目面试」**。
- 投递要求是**作品**：波起二象要「一个 AI 改造案例 + MVP 在线链接 + 运行截图 + 实测数据 + 客户反馈 + GitHub」。
- **结论：对入门级 FDE 岗，真实交付作品集 = 录用标准本身。仓桥的全部价值，就是批量制造这个作品集。**

**发现三：本体论是高薪天花板的钥匙。**
- 从 20-35K（波起二象，靠 Vibe Coding 作品）到 40-70K（字节/蚂蚁/灯塔，要本体/知识图谱/系统集成）的差距，**本质就是会议说的 L2→L3 的跃迁。仓桥的 V1-V7 必须把学生从「会搭 Agent」带到「会建本体」，这既是学生涨薪路径，也是仓桥交付能力的护城河。**

## 4. 解决方案：本体论驱动的 FDE 交付系统 + 双仓飞轮

仓桥的核心产品不是一个 App，是一套**「本体 + Skill + Agent + 人才」的 AI 落地操作系统**，跑在 Hermes 上。

### 4.1 项目仓 = 行业本体资产库（L3 落地）

每交付一个项目，沉淀：

- **行业本体（Ontology）**：该行业的统一业务地图——把客户分散系统的字段翻译成统一业务语言（如零售的「商品/库存/门店/会员」、茶空间的「茶品/茶旅/空间/会员包」），AI 基于本体理解业务，不碰原始字段，降幻觉、可复用。
- **Skill/Agent 模板**：爆款拆解、选题、客服、简历诊断、面经筛选、CRM 同步、质检……（career-coach 已有 17 工具、lufei 已有 11 Skill）。
- **交付方法 + 质量门 + 案例数据**：HITL、L3/L4 隔离、Eval 评测。

> 项目仓让每次定制交付变成下次标准化复制的素材——**这是对 L1（拼人月，零沉淀）的根本超越。**

### 4.2 人才仓 = FDE 交付军团（V1-V7）

| 等级 | 角色 | 能力（对齐 §3.2 JD） | 对应可投岗 |
| --- | --- | --- | --- |
| V1 | AI 操作员 | 会用模板/工具，Python+SQL 基础 | — |
| V2 | 工作流搭建者 | Vibe Coding，按 SOP 组 Skill/Agent | — |
| V3 | **FDE-Delta（技术侧）** | RAG/Agent/Eval 全链路，独立搭 PoC→上线，有可展示作品集+GitHub | **波起二象/智控医疗（经验不限）** |
| V4 | **FDE-Echo（业务侧）** | 进客户现场拆需求、建行业本体、客户沟通 | AI 创业/垂类 FDE |
| V5 | 解决方案架构师 | 抽象行业本体模板、定交付标准、系统集成/私有化 | 大厂/政企 FDE（35-70K） |
| V6 | 项目主理人/OPC | 独立获客+交付+复盘，OPC 一人公司 | 益企同款 ¥2000/天 |
| V7 | 行业/区域合伙人 | 带队、沉淀方法论、本体战略 | 灯塔级 |

### 4.3 桥接：AI 做重复，人做判断

需求进项目仓 → 拆任务包（定本体/数据源/Skill/验收）→ 按等级派人才仓 → AI 承担生成/检索/质检/CRM → 人负责本体设计、审美、客户、终审 → 结果回流，本体变厚、案例变多、模板变全。

## 5. 双层项目仓：小红书入口 + 企业本体出口

这是仓桥相对所有竞品（尤其益企）的结构创新——**用两层项目仓覆盖 FDE 成长全曲线：**

```
入口层 · 小红书 AI 运营项目仓（V1-V3，自造血 + 出 IP + 出作品集）
  学生 0 基础 → 用 Hermes 搭采集/拆解/文案/复盘 Agent → 交付真实商家账号
  本体雏形：小红书内容运营本体（选题/爆款/人设/转化）
  产出：可公开的账号数据 + GitHub + 客户反馈 = 波起二象/智控医疗的录用作品
        ↓ 筛人 + 攒作品 + 赚到第一笔
进阶层 · 企业数实融合项目仓（V4-V7，补本体硬工程）
  茶空间 → 本地商家 → 中小企业 AI 落地
  补：行业本体建模 + SQL/ETL + 系统集成 + Hermes 二次开发 + 私有化
  产出：对齐字节/蚂蚁/政企 JD 的企业级交付能力
        ↓
出口：进大厂 FDE(35-70K) ／ AI创业 FDE(20-50K) ／ OPC 接单驻场(¥2000/天)
```

为什么必须两层（诚实面对 [fde-bp-comparison.md](./fde-bp-comparison.md) 的结论）：**小红书项目仓门槛低、自造血、出 IP，但只训得到 Echo+轻 Delta，训不出大厂要的本体/SQL-ETL/系统集成。** 所以小红书是 on-ramp（让学生进得来、赚得到、攒得起作品），企业本体层才是 destination（把学生送到 35-70K 的线）。**小红书是别人没有的入口，本体是高薪必须的出口。**

## 8. 竞争格局与差异化

| 类型 | 代表 | 缺口 | 仓桥差异化 |

| FDE 派遣/FDC | **益企商创智库** | 只收割 5-10 年熟手、无 IP、难规模化 | **从 0 造学生 + 小红书自造血 + IP 飞轮 + 本体资产** |

**核心护城河（一句话）：竞品要么停在 L1/L2（人头/一次性交付），要么只卖技能无作品、只卖作品无技能；仓桥把 L3 本体资产化——每个行业的业务地图 + Skill/Agent 模板持续沉淀，用 Hermes 工具链规模化交付，用大学生军团降成本，用小红书 IP 低成本获客。本体越厚、人才越多、案例越多，交付成本越低，这是别人抄不走的复利。**

## 9. 技术架构

L1 Hermes runtime（多模型/CLI/TUI/桌面/消息网关/记忆/技能/定时/MCP/本地云端）→ L2 通用 SkillPack → L3 行业 Profile（本体 + Skills + Tools + 数据边界 + Kanban + Dashboard）→ L4 客户私有数据（raw/wiki/CRM/日志，永不进 git）。

部署：本地/云端常驻（腾讯云 CVM 7×24）/混合；HITL 人审；高风险输出禁止无依据生成。**本体层是技术核心——降幻觉、经验资产化、跨客户复用。**

## 10. 里程碑

### 0-90 天（PMV 验证期）
- 写定 FDE 教练 persona + fork career-coach → fde-coach-profile。
- 跑通 **1 个完整闭环**：1 个大三学生 0 基础 → 用 Hermes 交付 1-2 个真实小红书商家 → 产出作品集 + 数据 → 投递波起二象/智控医疗类岗拿到面试 OR 接单成交。**录成案例视频（路演核心弹药）。**
- 项目仓达 5-8 个付费商家账号（自造血验证）。
- 开第一期 V1-V7 训练营（¥2,980-6,980，10-20 人）。

### 3-6 个月
- 10 个标杆案例（含 ≥1 个企业本体项目）；100 人人才仓；3 个行业本体包。

### 6-12 个月
- 50+ 付费商家/企业项目；500 人人才仓（50 人可派单）；≥10 个学生出口实证（大厂面试/OPC 接单）；月收入 ≥¥50 万。

## 13. 风险与诚实披露

| 风险 | 应对 |
| --- | --- |
| 过度承诺「保进大厂」 | 红线禁止；大厂 5-10 年门槛是硬约束，只承诺作品+能力+派单+认证；OPC 接单兜底 |
| 小红书项目仓训不出企业级 FDE | 双层项目仓，V4-V7 必须有企业本体项目 + 补 SQL/ETL/系统集成 |
| 本体建模能力团队是否够 | 先从已沉淀的 lufei/career-coach 本体起步；企业级请外部专家客座，不硬编 |
| 第一个闭环跑不通 | 前 90 天全公司只聚焦 1 个闭环 |
| 小红书/数据风控 | 慢速、授权/公开数据、人工转发、L3/L4 隔离、HITL |
| 数据可信度（尽调会问） | 薪资来自一手 JD；市场规模/缺口/就业率类为公开口径，融资前抽样复核 5-10 家报价 + 3-5 个学员真实出口 |

## 14. 一句话收束

> **AI 落地的瓶颈从来不是模型，是从「会聊天」到「懂业务」的那层本体翻译，以及能把它一个客户一个客户落地的人。仓桥用 Hermes 工具链把行业本体和交付方法沉淀成资产（项目仓），用真实项目把大学生炼成 FDE 交付军团（人才仓），一头让招不到人的企业用得起 AI 落地，一头给年轻人一份既能进大厂（35-70K）也能 OPC 接单（¥2000/天）的真本事。本体越厚、人才越多、案例越多、交付越便宜——这条由 AI 工具链点燃、由大学生军团驱动、由行业本体护城的飞轮，就是仓桥要在中国 ToB 跑出来的东西。**
