## 0. 先纠正一个会议记录的硬错误：FDE 是什么

会议纪要把 FDE 写成「自由职业者/设计师」——**这是元宝会议助手的语音转写错误，必须纠正，否则整个定位会跑偏。**

**FDE = Forward Deployed Engineer（前进部署工程师 / AI 落地交付工程师）。**

这正是仓桥 BP 里人才仓 V3 等级的那个角色（见 BP §4.2 表格：「V3 / AI FDE / 能面对客户拆需求并交付 / 行业 Copilot 搭建、客户试运行」）。会议里 azan 反复说的「FDE 交付」「入职大厂」「AI+落地」「两条路径：进大厂 or 接单」——全部指向这个工程交付岗，而不是设计师。

为什么这个纠正价值千金：

- **设计师赛道**已经被路飞本人占住了（前滴滴/唯品会设计总监，15 年面试官），azan 再做就是抄袭 + 内部打架。
- **FDE 赛道是 2025-26 增长最快、最不饱和的 AI 岗位**（详见 §3），且对非顶尖学历友好（拼交付和沟通，不拼论文），天然适配「大三大四学生陪跑」。
- azan 自己的核心优势是**程序/工程**（会议里他自己强调「在程序领域的专业优势」），FDE 正好是工程岗，人岗匹配。

**一句话定位（建议）：**

> **「FDE 职业教练 azan——带大三大四学生从 0 到拿下 AI 落地交付岗（进大厂 or 接单变现），用真实企业级项目做你的简历。」**

### 2.2 career-coach-copilot——直接 fork 的「交付后台」

这套已经是生产级（405 测试全绿），fork 过来改垂类即可，预计 **2-3 周**可改造完成：

- **5 角色流水线**：Elon（意图分流）→ Jobs（服务诊断/简历画像）→ Linus（草稿生成，白文本无 markdown）→ Turing（6 维质检，默认拒绝）→ Bezos（30 分钟定时扫学员、生成待办）。FDE 版只需把 Jobs/Linus 的知识库换成 FDE 内容。
- **17 个工具**：crm-sync / review-dashboard / usage-analyzer / profile-builder / wiki-compiler / persona-distiller / clawbot-router 等，多数纯 Python、依赖轻，可直接复用；profile-builder 改造成「FDE 候选人画像（GitHub 项目 + 技术栈）」。
- **L1-L4 隐私边界**：L3（代码/课程/脱敏知识进 git）+ L4（学员数据/聊天/CRM 永不进 git）。FDE 版照搬。
- **HITL 红线**：AI 只出草稿，教练审核后用本人微信发出——保护「陪跑」的服务温度，也是合规底线。
- **交付形态**：云端常驻（腾讯云 CVM 7×24）+ 教练手机半自动转发。FDE 版照搬。

### 2.3 lufei-creator-profile——「内容工作室」可复用的 11 个 Skill

这是路飞的 Hermes L3 Profile，把内容运营拆成 11 个 Skill + 3 个 MCP 工具。FDE 教练自己的内容生产（自媒体矩阵）直接装这套 Profile：

## 3. FDE 赛道调研（为什么是现在、为什么是这个岗）

### 3.1 FDE 是什么、为什么 2024-26 爆发

- **定义**：嵌入客户技术环境、亲手写生产代码并对系统跑通负责的工程师——part engineer / part consultant / part solutions architect。区别于咨询（交报告）和售前（促成交），FDE 的成功指标是**系统在生产环境跑起来**。
- **起源**：Palantir 早期（2010s），内部代号「Deltas」；2016 年前 Palantir 的 FDE 数量甚至多于普通工程师。
- **2024-26 爆发的根因——部署鸿沟**：MIT NANDA 2025 报告，**95% 企业生成式 AI 试点没有可衡量的业务影响**。瓶颈不在模型，在落地。FDE 就是那座桥。
  - 招聘量：FDE 岗位 YoY **+1165%**（2025 vs 2024，1000+ 岗位样本）；2026 初统计 **39 家 AI 公司开放 224 个 FDE 岗**。
  - OpenAI 2024 组建 FDE 团队；**2026 年 5 月成立 The Deployment Company**（融资 $4B+，~$10B pre-money），收购 Tomoro（~150 FDE）。Anthropic 成立 $1.5B 企业 JV、扩张 Applied AI 团队。
  - **58% 的 FDE 岗在 11-200 人的成长期创业公司**——这正是「接单/自由交付」路径的现实基础。

### 3.2 FDE 在中国——岗位、雇主、薪资、缺口

- **对应岗位名**：解决方案工程师 / 交付工程师 / AI 落地工程师 / 大模型应用开发工程师 / AI 解决方案架构师。
- **谁在招**：字节（火山引擎、飞书）、阿里云、华为、百度、美团、小红书。字节 2026 校招计划 5000+ 人（R&D +23%）；阿里 7000+ offer，AI 岗 >60%。
- **薪资信号**：大模型应用开发 ~¥37,336/月；AI 解决方案架构师 top 年薪 ~¥150 万；大厂 2026 校招 AI 研发普遍 ¥40 万+；脉脉数据 **>50% 的 AI 应届岗月薪 >¥5 万**。入门 AI 运营 ~¥18,457/月（陪跑入门档的地板）。
- **缺口是真的**：大模型应用人才缺口 **300-400 万**；雇主反复抱怨「不缺会调模型的，缺能把 AI 真正用起来产生价值的人」——这正是「陪跑→入职/接单」承诺可信的基础。

### 3.3 竞品与空位

- **通用职业教练标杆**：张雪峰/峰学蔚来——志愿填报市场 2023 年 ~¥9.5 亿（vs 2016 的 7.3 倍）；产品 ¥599 录播 → ¥11,999 梦想卡 → ¥17,999 梦想卡 Plus，多省断货。**证明中国家长/学生愿为高赌注的职业指导付 ¥1-2 万**，且团队交付（非创始人亲自）能规模化。
- **AI 训练营/求职机构（包就业）**：传智 AI 就业班 ¥21,800-26,800（号称 93.5% 就业、均薪 ~¥10,500/月）；千锋 ~¥22,800。卖的是**通用大模型/算法**，commodity 化、拼价格、出口薪资 ¥10K 档。
- **空位**：**没有人占住「FDE / AI 落地交付」这个定位**。它(a) 是最不饱和、增速最快的 AI 路径；(b) 对非顶尖学生友好（拼交付+沟通）；(c) 自带「接单/freelance」兜底（58% 岗位在小创业公司）。**这个三角——细分岗 + 双出口（入职 OR 接单）+ 交付作品集背书——就是那条没人走的车道。**

### 3.4 需求侧：学生想要吗、企业招不到吗

- **学生侧**：2026 校招是 AI 抢人大战，但通杀的筛选门槛是**项目经验/实战**——「普通简历过不了初筛」「卡在人才库没有面试」。**教练的楔子 = 毕业项目本身就是那个缺失的简历资产。**
- **企业侧**：缺口 300-400 万，63% 企业争抢大模型应用人才；95% 试点失败的结构性问题短期不会消失，FDE 溢价预计持续到 2026-27。
- **接单兜底可信**：58% FDE 岗在小创业公司（门槛低）；另有活跃的大学生 AI 接单市场（小程序/API 封装/自动化/RAG 小项目）。

## 4. 个人品牌打造路径

### 4.1 人设：照搬路飞公式，换 FDE 垂类

路飞人设公式 = **真实资历背书 + 反鸡汤 + 讲具体公司具体岗位的内幕**。FDE 版：

- **身份锚点**：「带过真实企业 AI 交付项目的 FDE / 解决方案工程师」+「仓桥智能 AI 落地交付」+「带学生进大厂/接单」。
- **差异化语气**（抄路飞的「不做什么」）：不说「姐妹/宝宝/普通人逆袭」；只讲具体公司（字节/OpenAI/火山引擎）、具体岗位（FDE/解决方案工程师）、具体追问（「你这个 RAG 真能 scale 吗」）。
- **azan 的「20% 不可复制身份」怎么补**：路飞靠 15 年面试官，azan 没有面试官年限，但有**仓桥的真实企业级交付案例**（路飞项目、求职顾问 Copilot、茶空间——BP §6）。**把这些真实项目变成「我带学生做过的 FDE 级交付」就是 azan 的护城河**。这也回答了会议里他对「企业案例做教学素材」的设想——对，但要变成**亲历交付**而非二手案例。
- **会议待办落地**：「围绕 FDE 职业教练人设统一规划个人形象和内容」= 先写一份 `persona.md`（抄 lufei-xhs-wiki/persona.md 的结构），定死：身份、语气、禁用词、3 个内容支柱、爆款配方。
- **会议提到「删除他人视频统一形象」**——品牌一致性方向对，但**别删除有数据的真实内容**；正确做法是建主账号（azan 本人 FDE 教练）+ 矩阵账号分发，用 `creator-quality-gate` 守人设一致性。

### 4.2 内容体系：vol.X 系列 + 三支柱

直接复用路飞的 vol.X 编号系列（算法复利）+ 爆款拆解十维。三个内容支柱：

1. **FDE 沉浸式模拟面试 vol.X**（视频，主爆款形态）
   - 标题公式：`vol.X [公司] FDE/解决方案工程师 模拟面试（[系统设计/Agent实战/客户沟通]压力面）`
   - 钩子公式（抄路飞）：真实场景暖场 → 攻击式高压问题，如「我是某 AI 大厂 FDE 面试官——你说你会 RAG，那 chunk 切错了召回崩了你怎么排查？」
2. **项目经历/作品集深挖**（图文+视频）
   - 「你简历写'做过 AI Agent'，面试官追问 5 层你能扛住吗」——把学生的项目从「做过 X」逼到「为什么 X 值得讲」。
3. **FDE 单点干货拆解**（图文工具型，主收藏）
   - 「FDE 必会的 6 个能力清单」「大厂 AI 落地岗最容易露馅的 8 个细节」「0 基础到 FDE 的学习路线图」。

爆款阈值（抄路飞实用干货类）：收藏 ≥1000 / 点赞 ≥300 / 评论 ≥20。

### 4.3 渠道：线上矩阵 + 线下讲座（呼应会议待办）

- **线上**：小红书（引流/真实案例）→ 公众号（深度）→ 短视频（抖音/视频号分发）→ 私域（微信群）。整套用 lufei-creator-profile 的内容工作室生产。
- **线下**（会议待办「准备小蜜蜂、找观众」）：浙大/浙财等高校讲座——这同时是 BP §8.2 的「高校放大器」。
  - 「找线下观众」的正确解法：**不要为了凑观众而凑**。用第一批私域学员 + 高校社团合作做首场，**现场录制变成线上 vol.X 素材**（一鱼两吃，呼应会议里「增加现场观众提升互动」）。
  - 小蜜蜂等设备：是对的，但属于 BP §15「获客成本」里的小钱，先把第一场内容跑通再批量买。

## 5. 产品化路径：V1-V7 课程框架 + 价格阶梯

会议反复提到「复用 V1 到 V7 的课程框架」。lufei-creator-profile 里其实**没有**显式的 V1-V7 课程文件（它有的是人才仓 V1-V5 等级）——**所以 V1-V7 需要 azan 现在把它定义出来**。建议把 BP 的人才仓 V1-V5 等级直接长成 FDE 学生的成长阶梯：

### 5.1 V1-V7 学习路径（FDE 养成）

| 阶段 | 能力目标 | 对应 BP 人才仓 | 产出物 |
| --- | --- | --- | --- |
| V1 | Python + SQL + Git 基本功 | V1 AI 操作员 | 环境跑通、第一个脚本 |
| V2 | API 集成 + LLM 应用基础 | V1→V2 | 调通一个 LLM 应用 |
| V3 | RAG 全链路（切块/嵌入/向量库/重排/混合检索） | V2 工作流搭建者 | 一个可演示的 RAG 系统 |
| V4 | Agent 框架（LangGraph/CrewAI）+ Eval/可观测 | V2→V3 | 一个 Agent 应用 + 评测 |
| V5 | **客户级完整交付项目**（拆需求→搭建→部署→交接） | V3 AI FDE | **毕业作品 = 简历项目经验** |
| V6 | 模拟面试 + 需求沟通 + 客户对话演练 | V3→V4 | 通过 mock 面试 |
| V7 | 真实接单 or 大厂内推面试 + 入职/晋升跟踪 | V4 解决方案架构师 | **入职 offer 或 接单收入** |

> 关键：**V5 的毕业项目就是 §3.4 里学生缺的那个「简历项目经验」**，也是企业筛选最看重的实战证据。这是整个产品最硬的价值点——把 azan 的「陪跑」从口号变成可交付的简历资产。
