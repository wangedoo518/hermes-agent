# 杭州仓桥智能科技有限公司 · 商业计划书

> 早期路演版 · 用一条主线讲清楚三件事：**为什么做（价值）／ 凭什么是我们做 ／ 怎么做。**

---

## 一、我们想做什么

> 🔍 **产品定位：仓桥是一家 AI Native 的「FDE 人才工厂」——把真实的 AI 落地项目变成训练场，把大学生炼成"能把 AI 真正用起来"的交付工程师（FDE），一头供给招不到人的企业，一头让年轻人拿到既能进大厂、也能接单的真本事。**

我们做的不是又一个"AI 工具"，也不是又一个"培训班"。我们做的是一条流水线：**真实项目 → 训练人 → 人交付更多项目**。

### 看到的新趋势

1. **AI 不缺模型，缺"能把它用起来的人"。** 2025 年 MIT 的报告说，95% 的企业 AI 试点没产生可衡量的业务结果。卡点不在模型，在落地——而落地是由人扛的。这个人有个名字，叫 **FDE（Forward Deployed Engineer，前沿部署工程师）**，Palantir 首创、OpenAI/Anthropic 正在疯抢的岗位。

2. **这个岗位在中国正在爆发，而且对学生敞开了一道门。** 我们在 BOSS 直聘上扒了 19 个真实在招的 FDE 岗，亲眼看到两件事：
   - **大厂在抢人、给得起钱**：字节火山引擎 FDE **40-70K**、蚂蚁数科 **40-60K·15薪**、阿里云 **35-55K·13薪**。
   - **AI 创业公司不看年限、只看作品**：智控科技（医疗 AI，25-50K）写着"**不以工作年限为核心筛选标准，应届生欢迎，请携带你的 Vibe Coding 项目来面试**"；波起二象（20-35K）写着"**经验不限、学历不限、按能力与作品定级**"，投递要求是"一个你做过的 AI 改造案例 + 在线 MVP + 实测数据 + 客户反馈 + GitHub"。

3. **年轻人缺的恰好就是那个"作品"。** 2026 届毕业生 1270 万，创历史新高。所有人都卡在同一道门槛上：**没有真实项目经验，简历过不了初筛。**

> 一句话：企业在喊"招不到能落地 AI 的人"，学生在喊"我没有能证明自己的项目"，中间缺一座桥。**仓桥就是这座桥。**

## 二、为什么是现在做：第一性原理

前面讲了想做什么，这部分讲为什么这件事过去做不成、现在能成。

### 当前类似的需求，大家是怎么解决的？为什么都不够

| 谁在做 | 怎么做 | 为什么不够 |
| --- | --- | --- |
| 通用大模型 / Agent 平台 | 卖通用能力 | 不懂客户的私有业务，落地全靠人 |
| 传统咨询 / 外包 | 拼人月、驻场定制 | 贵、慢、易返工，干完零沉淀（国内 ToB 已被它做臭） |
| AI 训练营（黑马/千锋） | 脱产课，¥2-2.8 万 | 课纲驱动，**没有真实客户作品**，还背"包就业"的雷 |
| 求职咨询（职优你等） | 改简历、内推 | **不教技术**，退款纠纷多 |
| FDE 派遣公司（如益企商创智库） | 收 5-10 年熟手去驻场 | **大学生根本进不来**，没有自造血、没有流量 |

### 第一性原理：AI 落地的本质，是"本体翻译"

为什么 AI 落不了地？因为你让一个大模型直接面对企业几百张表、上千个字段，它必然看不懂、必然乱编。

真正的解法，是先建一张**"业务地图"（本体 / Ontology）**——把各个系统里零散的字段，翻译成统一的业务语言（"商品""仓库""会员""茶旅"）。AI 不碰复杂字段，只看这张地图，就能听懂业务、少出幻觉，而且**这张地图一旦建好，就是可复用的资产。**

> 🔍 **这就是 FDE 的"第三层认知"：FDE 不是拼人月（第一层），也不只是"咨询+开发"的复合角色（第二层），而是把一个行业的"业务地图"沉淀下来、反复复用的人。谁掌握了某个行业的本体，谁就掌握了这个行业 AI 落地的护城河。**

### 新变量：AI 工具链让"建地图 + 搭 Agent"变得很便宜

过去建本体、做定制 Agent 成本极高，所以 FDE 模式只有 Palantir 这种公司玩得起。现在 AI 编程（Claude Code / Codex / Cursor）+ Agent 工具链（我们用的 **Hermes**）把这个成本打下来了——**一个被训练过的大学生，配上 AI，就能干以前一个团队才能干的活。** 这是仓桥能成立的时间窗。

而且自然语言交互，恰好契合中国组织"靠人沟通、流程灵活"的习惯，AI 反而能在中国 ToB 跑得更顺。

## 三、FDE 交付的本质：四个环节，AI 在每一环都能发力

跟"交易"可以拆成几个环节一样，**一次 AI 落地交付也能拆成四环**。看懂这四环，就看懂了我们的产品在哪发力：

```
① 懂需求  →  ② 建本体  →  ③ 搭系统  →  ④ 沉资产
 (Echo)      (业务地图)    (Delta)     (可复用)
```

| 环节 | 在做什么 | AI / 仓桥怎么发力 |
| --- | --- | --- |
| ① 懂需求 | 进客户现场，把"老板想用 AI"翻译成能干的活 | 学生练的就是这个（Echo 角色），AI 帮做需求结构化 |
| ② 建本体 | 把客户黑话变成统一业务地图 | **仓桥的核心资产层**，越建越厚、跨客户复用 |
| ③ 搭系统 | 用 Hermes 搭 Agent、RAG、工具调用，上线交付 | AI 编程把搭建成本打下来（Delta 角色） |
| ④ 沉资产 | 把这次的活沉淀成模板、SOP、案例 | 下次交付直接复用，成本递减 |

> 这四环里，**②建本体 是别人没有的护城河，③搭系统 是 AI 工具链给的杠杆，而 ①④ 正好是大学生能上手、且越练越值钱的地方。**

## 四、凭什么是我们做

这是投资人最关心的问题。我们的答案是三句话：**已经有跑通的真实案例、已经有写好的交付后台、市场已经在点名招我们用的技术栈。**

### 1. 三套已经验证的真实资产（不是 PPT）

- **路飞设计求职辅导**：一个知识付费 IP 的完整跑通——小红书内容机器 + 公私域漏斗 + 成交（167 单、155 学员、369 次咨询）。这是我们的"内容获客 + 人设运营"模板。
- **刘一欣求职顾问 Copilot**：一套生产级 AI 交付后台——5 个角色的工作流、17 个工具、405 个测试全绿、严格的人审红线和数据隔离。这是我们的"交付工程"模板，可以直接复制到任何行业。
- **茶文化小红书项目**：验证了"AI 干高效率的活、人做审美和信任"的本地生活样板，也是学生的第一批真实练手客户。

### 2. 市场已经在为我们的技术栈点名

我们扒的 19 个 FDE 岗里，有公司明确招"**会 Hermes 二次开发**"的人，有公司要"**AI Agent Skill（技能组）开发与交付**"，腾讯云要"Agent、RAG、MCP、Harness"。**这些正是我们的学生每天在 Hermes 上做的东西——我们教的，就是市场逐条在招的。**

### 3. 团队背书

- **CEO（azan）**：核心优势在工程与 AI 落地，已亲手沉淀路飞、求职顾问、茶空间三套真实项目，是这套打法的设计者。
- **技术**：Hermes Agent 工具链的深度使用与二次开发能力（多模型、Skill、记忆、MCP、本地/云端部署）。
- **顾问 / 合作**：路飞类知识付费 IP、求职顾问团队、茶空间主理人、高校社群。

## 五、怎么做：双层项目仓 × FDE 人才仓

我们的做法可以用一张图讲清楚：**两个仓，一座桥。**

```
项目仓（资产越攒越厚）            人才仓（人越练越强）
  行业本体 + Skill 模板  ←—— 桥 ——→  大学生 → V1...V7 → FDE
  真实交付案例 + 数据                  AI 做重复，人做判断
```

### 双层项目仓：小红书做入口，企业本体做出口

我们诚实地承认：小红书运营门槛低、能自己赚钱、还能出内容引流，但它**练不出大厂要的企业级硬工程**。所以我们用两层：

- **入口层 · 小红书 AI 运营**（V1-V3）：学生 0 基础进来，用 Hermes 搭采集/拆解/文案/复盘的 Agent，交付真实商家账号（¥3,000-8,000/月）。**好处：门槛低、自造血、产出的账号数据+GitHub 正好是波起二象/智控医疗那种岗的"录用作品"。**
- **出口层 · 企业 AI 落地**（V4-V7）：在茶空间→本地商家→中小企业的真实项目里，补上本体建模、系统集成、Hermes 二次开发——对齐字节/蚂蚁那种 35-70K 的岗。

学生从这条线出来，有两个出口：**进大厂当 FDE（35-70K），或者自己接单/OPC 驻场（对标益企 ¥2000/天）。** 因为有"接单"这个兜底，我们**不卖"保进大厂"**，只承诺"真本事 + 真作品 + 派单机会"——这也躲开了培训行业最大的退款雷。

### 用户旅程

| 学生侧 | 企业侧 |
| --- | --- |
| 看小红书内容 → 进私域 → 报 ¥9.9 测评 | 在小红书看到案例 → 找上门 |
| 进 V1-V7 训练营，边学边接真实单 | 仓桥用 AI+学生交付，¥3-8K/月起 |
| 产出毕业项目（作品集 + 数据 + GitHub） | 项目越多，本体越厚，下次更便宜 |
| 出口：进大厂面试 / 接单变现 | 复购 + 效果分成 |

## 六、商业模式与增长策略

### 收入模型（三段，前期就有现金流）

- **项目仓收入（自造血）**：小红书代运营 ¥3-8K/月/账号；企业 AI 落地项目 ¥5-30 万/个；运维陪跑 + 效果分成。
- **人才仓收入**：¥9.9 测评 → ¥2,980-6,980 系统训练营 → ¥1-3 万 6 个月全陪跑。
- **平台化（远期）**：项目撮合佣金、本体/SkillPack 授权、合伙人分成。

### 增长策略：先单点闭环，再复制

我们不铺大盘子。**第一步只做一件事：在 90 天里，把"一个大三学生从 0 到拿下 AI 落地交付岗"这一个完整闭环跑通，并录下来。** 这个录像一鱼三吃：是个人 IP 的内容、是训练营的招生案例、是项目仓的第一块拼图。跑通 1 个，就能复制 N 个——用内容 IP 低成本获客（抄路飞），用高校讲座做线下入口。

## 七、发展里程碑

- **0-90 天**：跑通并录下第 1 个学生闭环；项目仓 5-8 个付费商家；开第一期训练营（10-20 人）。
- **3-6 个月**：10 个标杆案例（含 ≥1 个企业本体项目）；100 人人才仓；3 个行业本体包。
- **6-12 个月**：50+ 付费客户；500 人人才仓（50 人可派单）；≥10 个学生出口实证；月收入 ≥¥50 万。

## 八、需要的团队能力 & 当前团队

**需要补齐**：本体/数据建模专家、FDE 交付负责人（项目管理/质量门）、企业 BD、高校招生负责人。

**当前团队**：CEO（azan，AI 工程 + 三套真实项目沉淀）、技术（Hermes 架构与二次开发）、运营（小红书 IP + 线下空间）。

**融资**：天使轮 **300-500 万**，释放 10%-15%。用途：产品技术 40%、标杆闭环 25%、人才仓 20%、市场 10%、合规 5%。
